# governed-ai-coding-runtime

## 服务哪一层

- `governance-sidecar`
- gate / evidence / selector / closeout routing

## 什么时候先看它

- 当当前仓要升级 `planning truth`、selector、preflight、repo-level change-evidence、formal reference governance companion 时
- 当需要借鉴“如何把产品主线和治理增强面分层”时

## 什么时候不该扩大到它

- 当问题属于当前主线运行时实现、`runtime/host-orchestrator` 目录设计、AgentBridge 兼容 adapter 细节时
- 不要把它误当作当前仓 runtime truth，也不要把它变成默认每次都先刷新的主线参考仓
