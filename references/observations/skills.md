## 为什么保留

这个仓主要是为了以后吸收 `Hermes` 产出的 `skills-drafts/` 时，有一个公开可比照的结构和写法参考。

它的价值不在当前 bring-up，而在后续这些场景：

- skill 目录结构怎么组织
- `SKILL.md` 怎样写得可复用、可验证
- 什么时候应该把本地经验上升为可复用 skill

## 什么时候先看它

- 要把某条稳定运维经验沉淀成 skill 时
- 要判断某种 skill 结构是不是已经偏离主流做法时

## 什么时候不要扩大到它

- 当前 Hermes runtime 或 MCP 协议排查，不要先跳到这里
