# AgentBridge 中文操作说明

这份文档只讲最常用的本机操作：

- 如何录入 1 个 / 2 个 / 3 个 key
- 如何切换当前生效的 key
- 如何运行一次性 bring-up
- 这些 key 会不会落盘

如果你要看完整技术边界和阶段说明，仍以 [hermes-docker.md](C:/Users/sciman/Documents/AgentBridge/docs/hermes-docker.md) 为准。

## 1. 基本原则

这套设计默认：

- `Codex` 是 Windows 主力执行层
- `Hermes` 是隔离学习层
- key 不写进聊天
- key 不写进 `.env`
- key 不写进 repo 文件
- key 不写进 PowerShell profile
- key 只放在当前 PowerShell 会话里

也就是说，你每次开一个新的 PowerShell 窗口，录入本次要用的 key；用完就清掉，或者直接关窗口。

## 2. 现在的 key 模型

现在不是固定“主用 + 备用”两把 key。

现在的模型是：`slot 列表`。

每个 slot 都是一组：

- `slot 名称`
- `这个 slot 对应的 base_url`
- `这个 slot 对应的 key`

例如：

- `primary=https://provider-a.example/v1`
- `backup=https://provider-b.example/v1`
- `glm=https://provider-c.example/v1`

你可以：

- 只有 1 个 slot
- 同时有 2 个 slot
- 同时有 3 个 slot
- 以后继续加更多 slot

但任意时刻只会有 1 个 slot 是“当前生效”的。

## 3. 先进入目录

先在普通 PowerShell 里进入这里：

```powershell
Set-Location 'C:\Users\sciman\Documents\AgentBridge'
```

这一步通常不需要管理员权限。

如果你当前不在 `C:\Users\sciman\Documents\AgentBridge`，就不要直接抄 `.\scripts\...` 这种相对路径。

这时有两种安全写法：

1. 先切目录，再运行相对路径命令
2. 直接运行绝对路径脚本

例如：

```powershell
& 'C:\Users\sciman\Documents\AgentBridge\scripts\invoke-hermes-bringup-once.ps1' `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1'
```

## 4. 录入 key

如果你不想在终端里逐个输入 key，也可以把 slot 信息先写进某个 `.env` 文件，再通过 `-EnvFilePath` 读取。

推荐的标准写法是：

```dotenv
HERMES_SLOTS=primary,backup
primary_base_url=https://provider-a.example/v1
primary_key=你的主用key
backup_base_url=https://provider-b.example/v1
backup_key=你的备用key
```

如果这个 `.env` 还同时服务别的项目，里面可以继续存在别的环境变量。

现在脚本会把 `HERMES_SLOTS` 当成唯一权威清单，只读取这里声明过的 Hermes slot，不会再把别的项目变量误认成 Hermes key/base_url。

然后这样加载：

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action load `
  -EnvFilePath 'D:\CODE\qq-codex-bot\.env'
```

如果 `.env` 里已经同时包含 `base_url + key`，就不需要再额外传 `-Slots`。

### 4.1 只有 1 个 key

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action load `
  -Slots 'primary=https://provider-a.example/v1'
```

运行后，脚本会安全提示你输入：

- `primary` 对应的 key

### 4.2 有 2 个 key

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action load `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1'
```

运行后，脚本会依次安全提示你输入：

- `primary` 的 key
- `backup` 的 key

### 4.3 有 3 个 key

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action load `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1','glm=https://provider-c.example/v1'
```

运行后，脚本会依次安全提示你输入：

- `primary` 的 key
- `backup` 的 key
- `glm` 的 key

## 5. 查看当前状态

录入完成后，可以看当前会话状态：

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action status
```

它会告诉你：

- 当前活动 slot 是谁
- 当前活动 key 是否已加载
- 当前活动 base_url 是否已加载
- 当前会话里一共加载了哪些 slot
- 当前投影给 Hermes 的 `inference_model`
- 当前投影给 Hermes 的 `inference_provider`

它不会把真实 key 打出来。

## 6. 切换当前生效的 key

如果你已经录入了多个 slot，可以切换当前生效的那个：

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action switch -Slot backup
```

或者：

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action switch -Slot glm
```

切换时会一起切换这两个环境变量：

- `HERMES_PROVIDER_API_KEY`
- `HERMES_PROVIDER_BASE_URL`

所以不会出现“key 换了，但 base_url 没换”的情况。

## 7. 清掉当前会话里的 key

如果你不想继续保留这些 key，可以执行：

```powershell
.\scripts\manage-hermes-provider-session.ps1 -Action clear
```

这会清掉当前 PowerShell 会话里的：

- 所有 slot 的 key
- 所有 slot 的 base_url
- 当前活动 slot
- 当前活动 key/base_url

如果你直接关闭这个 PowerShell 窗口，也会自然失效。

## 8. 一次性 bring-up

如果你不想手工分几步做，可以直接用一次性 bring-up 启动器。

### 8.1 只有 1 个 key

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1'
```

### 8.2 有 2 个 key

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1'
```

如果你已经把 key/base_url 写进标准 `.env`，也可以直接这样：

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -EnvFilePath 'D:\CODE\qq-codex-bot\.env'
```

如果你当前不在 `AgentBridge` 目录，改用：

```powershell
& 'C:\Users\sciman\Documents\AgentBridge\scripts\invoke-hermes-bringup-once.ps1' `
  -EnvFilePath 'D:\CODE\qq-codex-bot\.env'
```

### 8.3 有 3 个 key

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1','glm=https://provider-c.example/v1'
```

它会自动做这些事：

1. 让你安全输入每个 slot 的 key
2. 重跑 `AgentBridge` 契约检查
3. 重跑 Hermes bring-up gates
4. 默认执行一次自动退出的 smoke 检查
5. 验证容器边界
6. 默认生成并校验 snapshot
7. 默认在结束后清掉当前会话里的 key

当前已经实测通过的最短命令是：

```powershell
& 'C:\Users\sciman\Documents\AgentBridge\scripts\invoke-hermes-bringup-once.ps1' `
  -EnvFilePath 'D:\CODE\qq-codex-bot\.env'
```

## 9. 常用变体

### 9.1 指定初始活动 slot

默认会先用 `primary` 作为活动 slot。

如果你想一开始就让别的 slot 生效：

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1','glm=https://provider-c.example/v1' `
  -Slot glm
```

### 9.2 如果你想进入交互式 chat

一次性 bring-up 启动器默认不会直接进入交互 chat。

这样设计是为了避免你输入完 key 以后，看起来像“卡住不动”。

如果你明确用了 `chat --cli`，终端又被占住但没有任何提示，不要先怀疑 key 或 provider。旧版 `run-hermes-wrapper.sh` 会把交互式 `hermes` 输出抓进 shell 变量，classic REPL 因为 `stdout` 不再是 TTY，看起来像“假死”。当前 live 脚本已经修复：交互式 `chat/auth/model/setup` 在检测到真实终端后会直接接管 TTY。

如果你就是想在 bring-up 过程中直接进入 Hermes 交互界面，请显式写：

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1' `
  chat
```

如果你当前不在 `AgentBridge` 目录，改用：

```powershell
& 'C:\Users\sciman\Documents\AgentBridge\scripts\invoke-hermes-bringup-once.ps1' `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1' `
  chat
```

如果你想强制 classic REPL，就写成：

```powershell
& 'C:\Users\sciman\Documents\AgentBridge\scripts\invoke-hermes-bringup-once.ps1' `
  -EnvFilePath 'D:\CODE\qq-codex-bot\.env' `
  chat --cli
```

当前修复后，它应直接显示 Hermes 经典 CLI；退出用 `/quit` 或 `Ctrl+C`。

### 9.3 不想自动清掉会话

如果你想 bring-up 之后继续保留当前会话里的 key，便于后续手工切换：

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1' `
  -KeepSession
```

### 9.4 暂时不做 snapshot

```powershell
.\scripts\invoke-hermes-bringup-once.ps1 `
  -Slots 'primary=https://provider-a.example/v1','backup=https://provider-b.example/v1' `
  -SkipSnapshot
```
