# Closeout Checklist

> Status: repo-owned reusable checklist template.
> Fill `closeout-bundle.example.json` first, then use this checklist as the final human closeout pass.

- [ ] 代码是否并回主线
- [ ] 相关测试是否通过
- [ ] verifier 是否通过
- [ ] selector 是否通过
- [ ] preflight 是否通过
- [ ] `closeout_bundle.json` 是否已更新并附上 evidence refs
- [ ] README / planning-status / backlog / plan / evidence 是否同步
- [ ] worktree 是否删除
- [ ] 临时 branch 是否删除
- [ ] 最终结论是否明确区分：
- [ ] `repo-side done`
- [ ] `platform/live still open`
